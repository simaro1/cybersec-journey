# [The OSI Model]

**Source:** Professor Messer Network+ (N10-009)
**Date:** July 22 2026

## The gist
The different levels in which a network system essentially functions off of. 
Used as a universal language, or guide, among professionals when referring to certain concepts. 

## Key points
Layer 1 is the physical layer where the signals go through the cables and fibers
Layer 2 is the data link where it allows devices to communicate with eachother. It's called DLC or MAC address on ethernet or wireless.
MAC addresses refer to hardware addresses on the DLC hardware
Layer 3 is the network layer which invovles IP address and communicating and being able to fragment frames to send them between networks
Stuff like subnetting or routing
Layer 4 is the transport layer thats also known as the post office layer. Being able to send info from one network to another. Protocols like TCP and UDP
which lets us take packets from one device to another.
Layer 5 is the Session layer that allows management to be able to talk from point a to point B. so starting stopping or restarting a session is this layer
Layer 6 involves putting all this info together so we can eventually see it. usually paired with layer 7. 
Layer 7 is the application layer, so stuff like HTTP, FTP, DNS, POP3 and all. 

## Example
Layer 1 would be cables and fiber. 
Layer 2 would be things like Mac address, frame, EUI-48, Switch
Layer 3 would be IP addresses, routers, and even packets
Layer 4 would be TCP segments, UDP datagrams
layer 5 would be control potocls
Layer 6 would be app encryptions (SSL/TLS)
Layer 7 would be what you see like HTTP DNS etc
